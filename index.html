Lesson 1: Basics of HTML

What is the web?

The Web is a network of computers that communicate with one another using the Hypertext Transfer Protocol (HTTP). When users request a webpage like (www.ivy-way.com) their browsers fetch the Hypertext markup language (HTML) document and display it. HTML documents can contain text, images, links, videos, and anything else that can be programmed.
Sketch of the Web by Steve Huffman for Udacity

HTML Elements, Tags, and Attributes

HTML stands for:
  • H: Hyper
  • T: Text (What you see)
  • M: Markup (How it looks)
  • L: Language
The Markup aspect of HTML is achieved by Elements, Tags, and Attributes. Elements are semantic structures that describe the content inside. Tags marks the start and end of an Element. Attributes define properties for an Element. For more informtion, click here.
https://familysearch.org/

Block vs. Inline Elements

Block elements start a new line of text and form invisible boxes around the text, which stretch as far as possible to the left and right. Some common examples of block elements are:
  • <h1>text</h1> = heading
  • <p>text</p> = paragraph
  • <div>text</div> = define a section
Inline elements format text without starting a new line. They take up the minimum amount of space necessary. Some common examples or inline elements are:
  • <b>text</b> = bold
  • <em>text</em> = italic
  • <a href="url"> = hyperlinks
  • <img href"url"> = images
Another important distinction is between Void and Non-Void elements. Void elements require only an opening tag. Non-Void elements require an opening and a closing tab. For more information on Block vs. Inline elements, click here.
Girldevelopit @ Github

Lesson 2: HTML Structure

A Website is like a House

One useful anology for understanding webpages is to think of them like a house. Consider...
  • HTML comprises the structure of a webpage. It's like the walls.
  • CSS styles the elements in a webpage. It's like paint and decorative trim.
  • Javascript controls the moving elements of a webpage. It's like the applicances you find inside a house.
This unit does not cover JavaScript, but for more informtion, click here.
https://teamtreehouse.com/

The Document Object Model (DOM)

The DOM is a convention for manipulating and viewing objects in HTML. In the DOM, pages are designed with a "Tree-Like Structure". Parent elements branch out into Child elements are that contained within the Parent. It is easy to explore the Tree-Like Structure of a webpage by opening Developer Tools in Chrome. For more information on the Document Object Model, click here.
University of Washington Computer Science Dept.

Boxify a Design

Often front-end web developers recieve PDF designs from graphic designers. The developers must turn these designs into websites. The first thing they do to accomplish this is Boxify. Here's how it works:
  • Everything on the web is made up of boxes, so draw boxes around everything you see that should be its own element.
  • Different people use different strategies. Some print a design and sketch boxes around elements with a pencil. Some cut out elements with scissors. Others use a sketch application on the computer.
  • Work from large to small, grouping smaller child elements inside larger parent elements.
After you Boxify a design, you will need to label each element with a class attribute. See the next topic for more information!
www.sendeasy.gr

Class Attributes

Once you Boxify a design, you need to label each of the elements. Some of them will fit into pre-made blocks like <h1> but most will need custom formatting. The most common way to do this is with Class Attributes. Class Attributes are like the labels on your boxes, so make sure they have names that make their purposes obvious.
http://klucid.com/

Lesson 3: CSS Style

Avoid Repetition

One of the beautiful things about CSS is that it helps programmers avoid repetition. Once you define a class with various styles, you can use that class again and again and again! Here's why that's so great:
  • You will be able to code faster.
  • If you make an error, you only need to correct it once.
  • Your website will automatically have consistency of appearance between similar elements.
Without CSS, your HTML document would be much longer!
www.coveybasics.com

Inheritance

Children inherit things from their parents (if they're lucky)! In HTML, properties of parent elements are passed on to child elements unless otherwise specified in the child class. For example, if we have:
  • 1   .parent {
  • 2       color: white;
  • 3       background-color: black;
  • 4   }
  • 5
  • 6   .child {
  • 7       background-color: red;
  • 8   }
then the text in both the parent and the child elements will be white, but their background colors will be different.

The Flexbox Technique

You may recall that block elements stretch horizontally to infinity and stack vertically. But what if you want to order elements horizontally? Use Flexbox! To order child elements horizontally within a parent element, add the following code to the parent:
  • 1   .parent {
  • 2       display: flex;
  • 3       display: -webkit-flex;
  • 4   }
Flex is powerful and does a lot more than just arrange elements horizontally. For more information click here.
www.w3.org

Code-Test-Refine

Programming is recursive. Take one step at a time until your project is complete! The instructors suggest this pattern:
  1. Boxify your design and plan the structure of the page.
  2. Look for repeated styles and label the elements.
  3. Write the HTML.
  4. Apply styles from biggest to smallest.
  5. Look at the details and fix what needs to be fixed until it is perfect.
